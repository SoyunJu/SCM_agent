# SCM Agent — 시스템 가정 사항

> 실제 외부 시스템 연동이 없는 경우 적용된 가정을 명시합니다.

## 이상징후 자동 처리

### OVER_STOCK / LONG_TERM_STOCK / SALES_DROP — 할인 판매 처리

**가정:** 안전재고 초과분(over_qty)이 최대 할인율 적용 시 당일 전량 판매된다고 가정합니다.

**이유:** 실제 쇼핑몰 판매 채널(상품 가격 변경 API 등)과의 연동이 없으므로,
판매 효과를 시뮬레이션하여 구글 시트 일별판매에 기록하는 방식으로 대체합니다.

**산출 방식:**
- `over_qty` = 현재재고 - 안전재고
- `max_discount` = (판매단가 - 매입단가) / 판매단가 × 100 (판매 이력 없으면 30% 기본값)
- `revenue` = 판매단가 × (1 - 할인율) × over_qty
- `cost` = 매입단가 × over_qty

**추후 개선 방향:** 실제 쇼핑몰 가격 변경 API 연동 시 해당 로직으로 교체 필요.
